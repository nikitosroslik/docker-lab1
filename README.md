# Лабораторная работа №1 — Основы Docker: контейнеризация .NET-приложения

## Структура репозитория

```
HelloDocker/   — консольное .NET-приложение в Docker
WebDocker/     — ASP.NET Core веб-приложение в Docker
```

---

## Контрольные вопросы

**1. Почему в Dockerfile для консоли используется `runtime`, а для web — `aspnet`?**

`runtime` содержит только базовую среду выполнения .NET — достаточно для консольного приложения.
`aspnet` — это `runtime` плюс библиотеки ASP.NET Core, без которых веб-приложение не запустится.

---

**2. Зачем нужен multi-stage build?**

Чтобы финальный образ был маленьким. SDK-образ (~700 МБ) нужен только для сборки.
В итоговый образ копируется только скомпилированный результат — финальный образ весит ~200 МБ вместо 700.

---

**3. Как передать переменную окружения через `docker run`?**

Флаг `-e`:

```bash
docker run -e ASPNETCORE_URLS=http://+:5000 web-dotnet
```

Несколько переменных:

```bash
docker run -e VAR1=value1 -e VAR2=value2 web-dotnet
```
